The Evaluate page ([`/evaluate`](https://meridian-udkd.vercel.app/evaluate)) scores a single token on demand using the deterministic version of the scout swarm — same rubric, same honesty rules, instant.

<Note>
  Evaluate is a **power tool for holders**, not the headline. Meridian's product is *discovery* (the daily shortlist). On-demand DD is the bonus.
</Note>

## What you get

For any pasted mint, the response includes:

<CardGroup cols={2}>
  <Card title="Composite score" icon="gauge">
    A 0–100 against the [rubric](scoring-rubric), with each sub-score broken out (`null` where Unknown).
  </Card>
  <Card title="Top reasons + standout risk" icon="list">
    Two strongest reasons it surfaced + the single biggest risk you should know.
  </Card>
  <Card title="Market metrics" icon="chart-mixed">
    Price, 24h change, liquidity, market cap, 24h volume, age, holders, buys/sells (1h).
  </Card>
  <Card title="Honesty signals" icon="shield-halved">
    Mint/freeze authority status, dev-wallet holding %, top-holders %, organic score, launchpad.
  </Card>
</CardGroup>

## How to use it

<Steps>
  <Step title="Paste a mint or pair address">
    Solana base58, 32–44 chars. The UI accepts either a mint or a DexScreener pair.
  </Step>
  <Step title="Hit Evaluate">
    The backend hits DexScreener + Solana RPC + Jupiter in parallel, then scores. Sub-second for most tokens.
  </Step>
  <Step title="Read the card">
    Composite + scout grid + market grid + reasons + risk + read. *Worth investigating — never financial advice.*
  </Step>
</Steps>

## API

```bash
curl -X POST https://meridian-backend-qae0.onrender.com/api/evaluate \
  -H "content-type: application/json" \
  -d '{"token":"G7L2LRZyoE6FZgFo51Betj88UPMdnNi1iYmBrpfpswrm"}'
```

### Error states

| Case | Response |
|---|---|
| Token not on Solana / not on DexScreener or Jupiter | `200 { "found": false, "error": "No Solana pair found for that token." }` |
| Empty `token` in body | `400 { "detail": "token required" }` |
| Backend cold-starting (Render free) | First request takes ~30s; subsequent are fast |

The endpoint **never throws on bad input** — it always returns a valid `EvaluateResponse` shape, so a frontend can render the error path cleanly.

## Embedding it

The endpoint is public and CORS-open. You can call it directly from a browser without a proxy:

```ts
const res = await fetch(
  "https://meridian-backend-qae0.onrender.com/api/evaluate",
  {
    method: "POST",
    headers: { "content-type": "application/json" },
    body: JSON.stringify({ token: mint }),
  }
);
const data = await res.json();
```
