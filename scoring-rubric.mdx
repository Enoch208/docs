The rubric is the contract. It's the same formula whether you're looking at the daily shortlist or the on-demand `/evaluate` — same scouts, same math, same honesty rules.

## Composite score

| Score | Tier | Read on the card |
|---|---|---|
| **75 – 100** | Strong | *Clean signals across the board — worth investigating.* |
| **55 – 74** | Mixed | *Mixed signals with upside — worth investigating with care.* |
| **0 – 54** | Weak | *Weak or unverified signals — investigate cautiously.* |

The composite is the **average of the available sub-scores** — Unknowns are *excluded*, never imputed as 0:

```
composite = mean(s for s in [onchain, liquidity, momentum] if s is not None)

if liquidity is None:
    composite = min(composite, 70)   # absence-of-evidence cap
```

That second line is important. Unverifiable liquidity caps the score at 70 — a token can't read as "clean" while its single biggest risk signal is missing.

## Sub-scores

See [the scouts](the-scouts) for the per-scout rules. In short:

| Scout | Range | Unknown when… |
|---|---|---|
| **On-chain** | 20 / 55 / 85 | Both mint + freeze authorities unverifiable |
| **Liquidity** | 0–100, scaled at $50k = 100 | No source has `liquidity_usd` |
| **Momentum** | 0–100, **volume-weighted** | No buy/sell ratio AND no volume AND no 24h change |
| **Smart-money** | **always None in v1** | The dedicated scout ships in v1.5 |

## Worked example — $MRDN

A live walkthrough on **$MRDN** (the Meridian token itself) from the on-demand `/evaluate`:

**Inputs (from DexScreener + Solana RPC + Jupiter):**

| Field | Value |
|---|---|
| `mint_authority` | `renounced` |
| `freeze_authority` | `renounced` |
| `liquidity_usd` | `$651` *(filled from Jupiter — DexScreener had `null`)* |
| `buys_h1 / sells_h1` | `0 / 0` *(genuine zero — bug 1 fix preserves this)* |
| `volume_h24` | `$651` |
| `price_change_24h` | `-50.4%` |

**Calculation:**

```
onchain   = 85           # both renounced
liquidity = round(651 / 50000 * 100) = 1     # thin
momentum:
  pressure   = 50 + (0 - 1) × 20  = 30
  pressure  += clamp(-25,+25, -50.4 / 4) = -12.6 → 17.4
  confidence = min(1.0, 651/25000)        = 0.026
  momentum   = 17.4 × (0.45 + 0.55 × 0.026) ≈ 8

composite = mean(85, 1, 8) ≈ 31
```

**Output:**

```json
{
  "composite_score": 31,
  "scores": { "onchain": 85, "liquidity": 1, "momentum": 8, "smart_money": null },
  "top_reasons": [
    "Mint & freeze authority renounced",
    "Thin liquidity $651"
  ],
  "standout_risk": "Thin liquidity — exit-rug risk",
  "one_line_read": "Weak or unverified signals — investigate cautiously.",
  "unknowns": ["smart_money"]
}
```

A 31 is honest: clean authorities, but real-thin liquidity and weak momentum on tiny volume. The rubric won't dress that up.

## What every pick carries

| Field | Example | Meaning |
|---|---|---|
| `composite_score` | `31` | The 0–100 overall |
| `scores.onchain` | `85` | Authority safety |
| `scores.liquidity` | `1` | Pool depth (Unknown if no source has it) |
| `scores.momentum` | `8` | Buy/sell pressure × 24h change × volume confidence |
| `scores.smart_money` | `null` | v1.5 |
| `top_reasons` | `["Mint & freeze renounced", "Thin liquidity $651"]` | Two strongest reasons |
| `standout_risk` | `"Thin liquidity — exit-rug risk"` | Always named on a positive pick |
| `one_line_read` | `"Weak or unverified…"` | Composite-aware plain-English |
| `unknowns` | `["smart_money"]` | Explicit list of gaps |
| `metrics.*` | (full market grid) | price, 24h, liquidity, mcap, vol, age, holders, top-holders %, dev %, auth, launchpad, socials |
